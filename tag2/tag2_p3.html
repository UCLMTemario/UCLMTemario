¿Qué es Docker?

Plataforma abierta para desarrollar, distribuir y ejecutar aplicaciones.

  • Separa las aplicaciones del resto de la infraestructura para que se pueda entregar software rápidamente.

  • Automatiza el despliegue de aplicaciones dentro de contenedores de software, proporcionando una capa adicional de abstracción.

¿Para quién es útil?

Útil para administradores de sistemas y para desarrolladores.

  • Para asegurar la calidadd e desarrollo tenemos que asegurar que todo el mundo usa las mismas versiones de todas las aplicaciones y librerías necesarios.

¿Qué es visualización?

Linux

No es virtualizado.

  • Los procesos dentro del contenedor de docker se ejecutan con el mismo kernel que la máquina anfitrión.

  • Aísla los procesos del resto del sistema.

  • Es capaz de controlar los recursos que se les asignan a los contenedores.

  • Internamente, el contenedor no sabe que lo es; como una distribución linux independiente, pero sin la penalización de rendimiento.

Windows

Utiliza tecnología de virtualización para ejecutar contenedores.

  • Docker Desktop for Windows utiliza Hyper-V para crear una maquina virtual ligera y luego ejecuta los contenedores dentro de está máquina virtual.

  • Proporciona un entorno Linux donde se ejecutan los contenedores Docker.

La plataforma Docker

Empaqueta y ejecuta una aplicación en un entorno aislado.

  • El aislamiento y la seguridad permiten ejecutar varios contenedores simultaneamente en un mismo host.

  • Los contenedores son ligeros y contienen todo lo necesario para ejecutar la aplicación.

Todas las personas con las que se comparte tienen el mismo contenedor que funciona de la misma manera.

  • Proporciona herramientas y una plataforma para gestionar el ciclo de vida de sus contenedores.

  • El contenedor se convierte en la unidad para distribuir y probar una aplicación.

Arquitectura

Docker Daemon

Escucha las peticiones de la API de Docker y gestiona objetos de Docker.

Docker Cliente

Interactúa con Docker.

  • Envía comandos al demonio docker, que los ejecuta.

  • El comando docker utiliza la API de Docker.

Docker Registries

Almacena imágenes Docker.

  • Docker Hub: registro público que cualquiera puede usar.

  • Es posible ejecutar en registro privado.

Comandos

  1. Docker pull/Docker run: Docker extrae las imágenes necesarias de tu registro configurado.

  2. Docker push: Docker empuja tu imagen a tu registro configurado.

Imágenes

Plantilla de solo lectura con instrucciones para crear un contenedor Docker.

  • Puedes crear tus propias imágenes o utilizar las creadas por otros y publicadas en un registro.

  • Ligeras, pequeñas y rápidas.

Dockerfile

Define los pasos necesarios para crear una imagen y ejecutarla.

  • Cada instrucción de un Dockerfile crea una capa en la imagen.

  • Cuando cambias el Dockerfile y reconstruyes la imagen, solo se reconstruyen las capas que han cambiado.

  1. FROM: especifica la imagen base sobre la que se construirá la nueva imagen

  2. RUN: ejecuta comandos en la imagen.

  3. CMD: define el comando que se ejecutará al iniciar un contenedor a partir de la imagen.

  4. ENV: establece variables de entorno dentro de la imagen.

  5. COPY/ADD: copia archivos desde el host al contenedor.

  6. VOLUME: define volúmenes para almacenar datos persistentes.

  7. WORKDIR: establece el directorio de trabajo dentro del contenedor para cualquier instrucción posterior.

  8. EXPOSE: expone puertos en el contenedor para que puedan ser accesibles desde fuera del contenedor.

  9. ENTRYPOINT: especifica el ejecutable predeterminado.

Contenedores

Instancia ejecutable de una imagen.

  • Se puede crear, iniciar, detener, mover o eliminar un contenedor mediante la API de Docker.

  • Por defecto, está relativamente aislado de otros contenedores y de su máquina anfitriona.

  • Está definido por su imagen y por cualquier opción de configuración proporcionada al crearlo o iniciarlo.

  • Cuando se elimina un contenedor, cualquier cambio en su estado que no esté almacenado en la memoria persistente desaparece.

Persistencia de datos

Si se elimina el contenedor, se eliminan también sus archivos.

Almacenar datos dentro de un contenedor

  1. A través de volúmenes.

  2. Montando un directorio de la máquina anfitrión al contenedor.

  3. Almacenándolo en la memoria del sistema, pero se perderían al reiniciar.

Docker Compose

Herramienta para automatizar la creación, inicio y parada de un contenedor o un conjunto de ellos.

  • Permite definir la configuración de los contenedores y sus interacciones en un único archivo YAML.

Comandos

  • Iniciar, detener y rehacer servicios.

  • Ver estado de los servicios.

  • Visualizar los logs.

  • Ejecutar un comando en un servicio.
