Tareas

  • Una aplicación puede constar de muchas tareas.

  • Las tareas se implementan como funciones.

  • Dos estados: ejecución y no ejecución.

El progreamador de tareas FreeRTOS es la única entidad que puede cambiar una tarea dentro y fuera.

Estados

  1. Una tarea puede ser preemptada debido a una tarea de mayor prioridad, porque ha sido retrasada o porque espera un evento.

  2. Listo: Cuando una tarea puede ejecutarse pero está esperando que el procesador este disponible.

  3. Bloqueada: Cuando una tarea está retrasada o está esperando a otra tarea.

  4. Suspendido: Una tarea entra y sale de este estado con vTaskSuspend(), vTaskResume() o xTaskResumeFromISR().

Colas

Las tareas tienen que comunicarse entre sí para que, colectivamente, puedan proporcionar funcionalidades útiles del sistema.

  • Una cola puede contener un número finito de elementos de datos de tamaño fijo.

  • FIFO: los datos se escriben en el extremo de la cola y se eliminan del frente de la cola.

Semáforo binario

Un semáforo binario puede ser utilizado para desbloquear una tarea cada vez que ocurra una interrupción particular, sincronizando efectivamente la tarea con la interrupción.

Permite que la mayoría del procesamiento de eventos de la interrupción se implemente dentro de la tarea sincronizada, con solo una porción muy rápida y corta que permanece directamente en el ISR.

Secciones críticas

Una porción del código necesita ser protegida de cualquier cambio de contexto para evitar que un cálculo se corrompa o que una operación de entrada/salida se corte o se mezcle con otra.

  • FreeRTOS protege de cualquier cambio de contexto, ya sea de una operación de planificación o de un evento de interrupción; o evita que cualquier tarea interrumpa en la sección crítica, pero permite que las interrupciones hagan su trabajo.

Semáforos mutex

Un mutex es un tipo especial de semáforo binario que se utiliza para controlar el acceso a un recurso que es compartido entre dos o más tareas.

  • Un token asociado con el recurso que se comparte.
