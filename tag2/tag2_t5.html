Introducción

¿Cuál crees que es uno de los principales retos del IoT?

La extracción de valor de los datos generados/recopilados.

  • La creación de cantidades masivas de datos es uno de los mayores retos desde el punto de vista de transporte y de gestión.

El valor real del IoT no reside únicamente en conectar cosas, sino en los datos que producen esas cosas, los nuevos servicios que se pueden habilitar a través de esas cosas conectadas y las perspectivas de negocio que pueden revelar los datos.

  • Análisis de datos: Analizar esta cantidad de datos de la manera más eficiente posible, ofreciendo perspectivas y conocimiento.

  • No todos los datos son iguales.

Big Data

Datos que son tan grandes, rápidos o complejos que es dificil procesarlos con los métodos tradicionales.

  • Proporciona respuesta a muchas preguntas que ni siquiera sabían que tenían.

  • Identifica problemas de forma más rápida y comprensible.

  • Detecta tendencias dentro de los datos.

  • Ayuda a aprovechar sus datos y utilizarlos para identificar nuevas oportunidades.

  • Gran herramienta para tomar mejores decisiones de negocio.

V’s

  1. Volumen: la organización recopila datos de diversas fuentes.

  2. Velocidad: los datos llegan a una velocidad sin precedentes y deben ser manejados en tiempo real.

  3. Variedad: los datos son de todo tipo de formatos.

  4. Variabilidad: los flujos de datos son impredecibles, cambian a menudo y varían mucho.

  5. Veracidad: calidad de los datos.

Tejido de datos

Conjunto de estrategias, técnicas y herramientas utilizadas para conectar, integrar y administrar datos de diferentes fuentes para obtener información valiosa.

  • Crea estructuras de datos sólidas y unificada a partir de datos dispersos y desordenados.

Especialmente importante en el contexto de IoT ya que un gran número de dispositivos y sensores generan un gran volumen de datos y es necesario tener una estructura sólida para procesarlos y obtener información valiosa.

Con la llegada del IoT, hay un mayor número de objetos y dispositivos conectados a Internet que generan datos sobre patrones de uso de los clientes y el rendimiento de los productos, pero el surgimiento del aprendizaje automático ha producido aún más datos.

Categorización de los datos

Datos estructurados

Los datos siguen un modelo o esquema que define cómo se representan un organizan los datos.

  • Se formatean, almacenan, consultan y procesan fácilmente.

Datos no estructurados

Carecen de esquema lógico que permita comprenderlos y descodificarlos con los medios tradicionales.

  • EL 80% de los datos de la empresa son no estructurados.

  • Métodos de análisis: Computación cognitiva y el aprendizaje automático.

Datos semi-estructurados

Categoría intermedia entre los datos estructurados y no estructurados.

  • No son relacionales, pero contienen un cierto esquema y coherencia.

  • Contienen metadatos que proporcionan información adicional sobre el contenido del dato.

  • Su procesamiento requiere una combinación de técnicas.

Análisis de datos IoT

Tipos

  1. Descriptivo: informa de lo que está ocurriendo, ya sea ahora o en el pasado.

  2. Diagnóstico: cuando se está interesado en el “por qué”.

  3. Predictivo: ayuda a prever problemas o iniciativas antes de que se produzcan.

  4. Prescriptivo: recomienda soluciones para problemas venideros.

  • El análisis predictivo y prescriptivo consume más recursos y aumenta la complejidad, pero el valor que aportan es mucho mayor.

Retos

Problemas de escalabilidad

Debido al gran númoer de objetos inteligentes en la mayoría de las redes IoT que envían datos continuamente, las bases de datos relacionados pueden crecer increiblemente rápido.

  • Problemas de rendimiento.

Volatilidad de los datos

  • Es posible que el modelo de datos cambie y evolucione con el tiempo.

  • Esquema dinámico.

Con las bases de datos relacionales, es fundamental que el esquema se diseñe correctamente desde el principio. Cambiarlo después puede ralentizar o impedir el funcionamiento de la base de datos. Debido a la falta de flexibilidad, las revisiones del esquema deben mantenerse al mínimo.

Machine learning

Diferencias entre Inteligencia Artificial, Machine Learning y Deep Learning

  • Inteligencia Artificial: campo que estudia cómo crear programas informáticos con la habilidad de aprender y razonar como los humanos para resolver problemas de forma creativa.

  • Machine Learning: Aplicación de la IA dedicada a la creación de algoritmos que permitan a los sistemas aprender sin intervención humana.

  • Deep Learning: Subconjunto de ML enfocado en la creación de redes neuronales artificiales.

Aspectos básicos

Rama de la inteligencia artificial que se enfoca en el desarrollo de algoritmos y técnicas que permiten a las computadoras aprender a partir de datos.

  • Su objetivo es desarrollar sistemas que puedan mejorar su rendimiento automáticamente a medida que se les presentan nuevos datos.

  • Tecnología clave el el ámbito del IoT ya que permite a los dispositivos tomar decisiones automatizadas y mejorar su rendimiento.

Da sentido a los datos que se generan, ya que muchos de estos pueden parecer incomprensibles/inservibles a simple vista.

Tipos

Aprendizaje supervisado

El algorimo se entrena con datos etiquetados.

  • El objetivo es que sea capaz de predecir la salida deseada para nuevas entradas no vistas anteriormente.

Aprendizaje no supervisado

El alboritmo se entrena con datos no etiquetados.

  • El objetivo es detectar patrones y relaciones en los datos.

  • Útil para explorar y entender los datos.

Aprendizaje por refuerzo

El algoritmo aprende a través de la interacción con su entorno y mediante la retroalimentación.

  • El objetivo es maximizar una recompensa y minimizar un castigo.

  • Utilizado para problemas de toma de decisiones.

Deep Learning

Técnica de ML que se basa en el uso de redes neuronales profundas para resolver problemas complejos.

  • Tienen varias capas de procesamiento, lo que les permite aprender características abstractas y complejas a partir de los datos.

“El Deep Learning anhela el Big Data porque es necesario para aislar patrones ocultos y para encontrar respuestas sin sobrecargar los datos. Con el Deep Learning, cuantos más datos de buena calidad tenga, mejores serán los resultados”

Tipos

  • Redes neuronales feedforward: son las más comunes y se utilizan para problemas de clasificación y regresión.

  • Redes neuronales recurrentes: se utilizan para problemas de procesamiento de secuencias.

  • Redes neuronales de refuerzo: se utilizan para problemas de aprendizaje por refuerzo.
