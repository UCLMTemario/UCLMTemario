Procesamiento inteligente

Inteligencia artificial

Disciplina científica que se ocupa de crear programas informáticos que ejecutan operaciones comparables a las que realiza la mente humana, como el aprendizaje o el razonamiento lógico.

Modelo de IA

Definición de reglas utilizadas para producir un resultado dada una entrada específica. Muchos funcionan como cajas negras.

Deductiva

Basado en reglas y premisas establecidas explícitamente. - Deducción: método por el cual se procede lógicamente de lo universal a lo particular.

Inductiva

Basado en reglas establecidas implícitamente. - Inducción: método por el cual se procede lógicamente de lo particular a lo general.

Aprendizaje automático

Rama de la inteligencia artificial cuyo objetivo es desarrollar técnicas para que las computadoras “aprendan”.

Aprendizaje no supervisado

Uso de datos no etiquetados (o etiquetados, no influye para el proceso) para entrenar algorítmos que identifican patrones y estructuras por sí solos. - Perfilar o segmentar clientes. - Detectar anomalias.

Aprendizaje supervisado

Uso de datos etiquetados para entrenar algorítmos que clasifican los datos o predicen los resultados con precisión. - Identificar tendencias en las ventas. - Optimizar cadenas de suministro y niveles de inventario.

Aprendizaje por refuerzo

Uso de datos para entretan algorítmos que resuelve problemas mediante prueba y error, recibiendo recompensas o penalizaciones basandose en la decisión elegida.

Retos

  • Eficiencia
  • Técnicas de optimización
  • Calidad de los datos
  • Sesgos
  • Ética

El flujo de datos

Tipos de flujo

Todos los flujos pasan por la aplicacion y la BBDD no tiene autonomia para iniciar un flujo. - Entrada: Usuario → Aplicación → BBDD. - Salida: Aplicación → BBDD → Aplicación → Usuario. - Retroalimentación al usuairo: Usuario → Aplicación → Usuario. - Procesamiento automático: Aplicación → BBDD → Aplicación → BBDD.

Fases del flujo en la aplicación

  • Carga: entrada de los datos de una fuente externa a la aplicación.
  • Transformación: adecuación de los datos a la siguiente etapa.
  • Procesamiento: serie de operaciones principales que se aplican sobre los datos.
  • Transmisión: envío de los datos a sud estino final.

Ahora sí, ¿cómo procesamos los datos de manera inteligente?

Parámetros

Valores internos de los modelos que establecen como transformar una entrada en una salidad siguiendo el algoritmo que tuviera asignado. - El usuario no puede modificar los parámetros. - Se establecen durante el proceso de aprendizaje.

Hiperparámetros

Valores que podemos establecer como usuarios que condicionan el aprendizaje de los modelos.

Distancias

  1. La distancia entre un punto y sí mismo es siempre 0.
  2. La distancia entre puntos diferentes es siempre positiva.
  3. Simetría: la distancia de p0 a p1 es la misma que de p1 a p0.
  4. Tiene que satisfacer la desigualdad de triángulos.
  • Distancia euclídea: es la más usada para describir distancias.
  • Distancia Manhatan: útil en espacios discretos.

Clustering

Consiste en descubrir grupos de puntos cercanos dentro de los datos. Los resultados pueden cambiar mucho entre diferentes algoritmos.

K-Means DBSCAN
¿Qué es? Basado en centroides: los valores más cercanos a cada centroide pertenecen a su cluster Basado en densidad: un cluster es una región con muchos puntos cercanos
Ventajas Fácil de interpretar Busqueda de outlier
Desventajas Clusters redondos Díficil de paramétrizar

Clasificación

Consiste en descubrir cuáles son las normas subyacentes detrás de una agrupación predefinnida dentro de los datos. - Clase: conjunto de valores discretos.

K-Nearest Neighbours Árboles de decisión
¿Qué es? Basado en similitudes con los vecinos del conjunto de datos original Basado en la asignación de clases en base a decisiones binarias
Ventajas Fácil de implementar y explicar Funcionan con todo tipo de datos
Desventajas Requiere guardar el conjunto de datos entero, escalando mal con muchos valores Díficil de parametrizar correctamente

Regresión

Consiste en descubrir las normas que relacionan valores continuos a un conjunto de datos.

Lineal
¿Qué es? Basado en modelos matemáticos polinómicos de grado menor a 2
Ventajas Puede extenderse a grados mayores con preprocesamiento
Desventajas Sensible a outliers

Consideraciones para elegir algoritmo

Machine learning en SIE

Beneficios

  • Mejor toma de decisiones.
  • Aumento de eficiencia operacional.
  • Mayor flexibilidad y adaptabilidad.
  • Mayor automatización.
  • Alta optimización de recursos.

Consideraciones en SIE

  • Eficiencia:
    • Los sistemas transaccionales deben funcionar en tiempo real.
    • Los modelos de IA suelen ser bastante lentos.
  • Calidad del dato: ocurren más errores si la introducción es manual y de muchas fuentes.
  • Heterogeneidad de los datos: es díficil construir modelos universales.
  • El procesamiento es fundamental.
  • Empleados humanos:
    • Resistencia al cambio.
    • Despidos y desplazamientos.
