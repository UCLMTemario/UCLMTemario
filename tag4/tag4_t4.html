Integración de Datos

Métodos de Integración

APIs

Especificación formal para definir la comunicación entre componentes software.

Servicios Web

Conjunto de aplicaciones o de tecnologías con capacidad para interoperar en la web y con el fin de proporcionar unos servicios.

SOAP

Protocolo simple de acceso a objetos.

  • Estándar de W3C.

  • Mensajes SOAP como XML.

WSDL

Lenguaje descriptivo de servicio web.

  • Estándar de W3C.

WSDL define la interfaz y el contrato del servicio web, especificando las operaciones disponibles y cómo interactuar con ellas.

Servicio REST

Cualquier interfaz entre sistemas que use HTTP para obtener datos o generar operaciones sobre esos datos.

  • Necesidad de simplicidad y flexibilidad.

  • Evolución de la arquitectura web.

  • Protocolo cliente/servidor sin estado.

  • Operaciones: GET, POST, PUT, DELETE.

  • Manipulación a través de URI.

  • Interfaz uniforme.

  • Arquitectura jerárquica entre componentes.

Servicio GraphQL

SIN TERMINAR

Almacenamiento basado en ficheros de texto plano

CSV

Formato no estandarizado abierto sencillo para representar datos en forma de tabla.

  • Principal uso: intercambiar datos de una base de datos relacional.

  • No hay una estructura estándar, por lo que permmite una gran variedad de interpretaciones de cómo es.

  1. Cada registro es una línea independiente, separada por un salto de línea.

  2. La última línea no tien por qué tener un salto de línea.

  3. La primera línea puede contener los nombres de los campos que contiene cada registro.

  4. Cada campo se separa por una coma.

  5. Cuando hay contenido de los campos especiales, debe estar marcado con la ““.

XML

Meta-lenguaje extensible de etiquetas desarrollado por W3C.

  • Principal uso: para almacenar e intercambiar datos.

Estructura jerárquica

Estructura de árbol.

  1. Comienza con un Elemento Raíz y ramas de la raíz a los hijos.

  2. Todos los elementos pueden tener sub-elementos.

Extensible

Un elemento puede ser añadido cuándo se necesite.

Autodescriptiva

Formato legible para el humano.

  • Las etiquetas describen el contenido del elemento.

JSON

Formato ligero de intercambio de datos.

  • Establece una estructura de datos legible y simple.

  • Se utiliza principalmente para transmitir datos entre servidor y aplicación web, como alternativa al XML.

Estructura jerárquica

  1. Colección de pares nombre-valor (objeto).

  2. Lista ordenada de valores (array).

Tipos de valor

  1. String.

  2. Number.

  3. Object.

  4. Array.

  5. True.

  6. False.

  7. Null.
